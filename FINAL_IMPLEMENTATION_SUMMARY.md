## 🚀 使用方法

### 对学习者
1. 打开算法页面（如冒泡排序）
2. 看到左边动画，右边代码
3. 点击播放按钮，代码行自动高亮
4. 切换 C++/Python 看不同语言的实现
5. 窄屏时自动变为竖排，或点击按钮手动切换

### 对开发者：为新算法添加支持

```javascript
// 1. 在算法文件顶部添加注释（可选）
// C++ code line mapping (1-indexed):
// Line 3: for (int i = 0; i < n - 1; i++)
// ...

// 2. 在步骤中添加代码行号
steps.push({
  // ... 其他字段
  cppLine: 3,      // C++ 代码第 3 行
  pythonLine: 3,   // Python 代码第 3 行
  description: '...',
})

// 3. 测试验证
// - 打开页面，确认代码行高亮正确
```

## 🔄 数据流

```
算法执行
  ↓
生成步骤 { array, comparing, cppLine, pythonLine, ... }
  ↓
useStepController 管理当前步骤
  ↓
useStepPublish 发布到 StepContext
  ↓
InteractiveVisualization 读取步骤
  ↓
┌─────────────────┬──────────────────┐
│                 │                  │
VizComponent   CodeBlock
(左侧动画)      (右侧代码)
    ↓              ↓
数组变化      高亮代码行
```

## 💡 关键特性

### 1. 双向绑定
- 步骤改变 → 动画更新 + 代码高亮
- 一个源头：`steps` 数组

### 2. 智能响应式
- 自动检测屏幕宽度
- 自动在 1400px 处切换布局
- 可手动覆盖（竖排按钮）

### 3. 灵活的代码映射
```javascript
// 三种方式都支持：
{ cppLine: 5, pythonLine: 5 }           // 方式 1
{ codeLines: { cpp: 5, python: 5 } }    // 方式 2
{ cpp: 5, python: 5 }                   // 备用
```

### 4. 向后兼容
- 没有代码行号的步骤不会崩溃
- 可逐步为算法添加支持
- 旧算法继续正常工作

## ⚙️ 技术细节

### InteractiveVisualization 组件
- 使用 Grid 布局（1fr 1fr 或 1fr）
- Sticky 定位代码框（不在窄屏）
- React Hooks：useState, useEffect, useStepData
- 屏幕宽度检测和响应式切换

### 代码行高亮流程
1. 读取 `stepData.current.cppLine` 或 `.pythonLine`
2. 传给 `CodeBlock` 的 `highlightLine` prop
3. CodeBlock 查找对应行元素
4. 添加高亮样式并滚动到视图（带平滑动画）

### 性能优化
- 使用 `useSyncExternalStore` 避免不必要重渲染
- Sticky 定位减少重流
- 步骤对象复用（不每次都创建新对象）

## 📞 问题排查

### 代码行没有高亮？
1. 检查步骤对象是否有 `cppLine`/`pythonLine` 字段
2. 检查行号是否正确（1-indexed，对应代码块的实际行）
3. 查看浏览器控制台是否有错误

### 布局没有变成并排？
1. 检查算法类型是否在 `VIZ_WITH_CODE` 集合中
2. 检查算法是否有 `code` 字段
3. 检查浏览器窗口宽度（应 > 1400px 才默认并排）

### 代码滚动不平滑？
1. 检查 `noAutoScroll` 参数是否正确
2. 查看是否有其他样式冲突
